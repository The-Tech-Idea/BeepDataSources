Editor & orchestration

How IDMEEditor fits around datasources loaded from this repository. Full detail: BeepDM/.cursor/beepdm/SKILL.md.

Core types

  • IDMEEditor — orchestration: datasource creation, lookup, helpers, ETL, mapping, unit-of-work.
  • IDataSource — runtime contract for CRUD, metadata, scripts, transactions (summary).
  • IDataSourceHelper — dialect-aware SQL/schema helpers.
  • ConfigEditor — persisted connections and metadata (summary).

Routing work

Use the narrowest skill: DI startup → beepserviceregistration or beepservice; building definitions → connectionproperties; validation → connection; CRUD → idatasource.

Datasource help in this repo

Static HTML groups plugins by family on Implementations (phased). REST/SaaS vendors document HTTP command surfaces on flagship conn-* pages (REST overview).

Plugin assemblies

Datasource projects in this repository are loaded at runtime through BeepDM discovery (for example AssemblyHandler / shared-context flows). See BeepDM/.cursor/assemblyhandler and related skills when changing load paths or NuGet driver layout.
