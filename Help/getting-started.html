How to use this help

Reading order, relationship to BeepDM skills, and how phased provider docs will land.

Open locally

Open Help/index.html in a browser, or serve the Help folder with any static file server. Scripts are plain ES5 classes; no build step is required.

Suggested reading order

  1. Editor & orchestration — where IDataSource instances live in an app.
  2. BeepService startup — desktop/bootstrap patterns.
  3. ConfigEditor — persisted connections and metadata.
  4. ConnectionProperties — building a definition.
  5. Connection lifecycle — validation, normalization, masking.
  6. IDataSource contract — CRUD, metadata, errors.
  7. Local & in-memory — SQLite, LiteDB, DuckDB (phase 03).
  8. RDBMS (SQL Server, PostgreSQL, MySQL, Oracle) — shared RDBSource / helpers (phase 04).
  9. NoSQL / document / KV / TSDB — MongoDB, Redis, RavenDB, CouchDB, InfluxDB (phase 05).
  10. Cloud & analytics — BigQuery, Snowflake, Spanner, Kusto, Presto (phase 06).
  11. Messaging & vector — Kafka, RabbitMQ, NATS, MassTransit, Redis Streams, Pub/Sub; Qdrant, Milvus, ChromaDB (phase 07).
  12. REST / SaaS connectors — Connectors/ by category, WebAPIDataSource pattern (phase 08); phase 09 flagship conn-* index table.
  13. Provider rollout — all phases.

BeepDM skills (authoritative)

This HTML summarizes behavior; the canonical guides are markdown skills under BeepDM/.cursor/ (e.g. idatasource, connection, configeditor). When instructions conflict, prefer the BeepDM repo and code.
