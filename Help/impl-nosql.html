NoSQL, document, and key-value

Cores under DataSourcesPluginsCore that register with DatasourceCategory.NOSQL. They implement IDataSource directly (not RDBSource); behavior differs widely from relational drivers.

Expectations vs RDBMS

  • Schema & metadata — “Tables” may be collections, keys, buckets, or indexes. Entity lists might be discovered lazily or approximated from samples.
  • SQL — There is no universal SQL layer; ad hoc query support is provider-specific (e.g. Flux for InfluxDB).
  • Migrations & DDL — RDBMS-style migration helpers in BeepDM often do not apply; prefer provider APIs and contract methods the core actually implements.
  • IDataSourceHelper — Unlike RDBMS, there is typically no shared dialect helper tree; logic lives in each core.

Contrast: REST / SaaS connectors

SaaS REST drivers under Connectors/ use DatasourceCategory.Connector, WebAPIDataSource, and WebAPIConnectionProperties — not document/KV wire protocols. See REST / SaaS overview and flagship conn-* pages.
