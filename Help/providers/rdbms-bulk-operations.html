Bulk Operations

High-performance batch insert and update strategies in RDBSource.BulkOperations

Overview

RDBSource.BulkOperations.cs (1,080 lines) provides optimized batch data operations that are 10-100x faster than row-by-row inserts. The implementation auto-selects the best strategy based on database capabilities.

Strategy Selection

StrategyMethodEnginesPerformance
Multi-Row INSERTBulkInsertMultiRow<T>SQL Server, MySQL, PostgreSQL, SQLite🚀 Fastest — single statement with multiple value tuples
Temp-Table MERGEBulkUpdateWithTempTable<T>SQL Server, MySQL, PostgreSQL🚀 Fast — bulk load into temp, single MERGE/UPDATE JOIN
Batched FallbackBulkInsertBatched<T>Oracle, DB2, all others🐢 Slower — individual statements in transaction

Usage

Bulk Insert

var orders = new List<Order> { /* 10,000 records */ };
await rdbSource.BulkInsertEntitiesAsync(orders, cancellationToken);
// Auto-selects multi-row INSERT for SQL Server/MySQL/PostgreSQL/SQLite
// Falls back to batched inserts for Oracle

Bulk Update

await rdbSource.BulkUpdateEntitiesAsync(updatedOrders, cancellationToken);
// SQL Server: creates #temp table → MERGE
// MySQL: creates TEMPORARY TABLE → UPDATE JOIN
// PostgreSQL: creates TEMP TABLE → UPDATE FROM
// Oracle: batched individual UPDATEs in transaction

Oracle Note

Reduce DefaultBatchSize below 1000 for Oracle to avoid ORA-01795 (max 1000 expressions). Oracle uses the batched fallback — multi-row INSERT is not supported.

Temp-Table Strategy (Updates)

For bulk updates, the temp-table strategy is:

  1. Create temp table matching target schema
  2. Bulk insert all updated rows into temp table
  3. Execute MERGE (SQL Server) or UPDATE JOIN (MySQL/PostgreSQL) from temp→target
  4. Drop temp table
EngineTemp Table SyntaxMerge/Update Syntax
SQL ServerCREATE TABLE #temp (...)MERGE target USING #temp ON ... WHEN MATCHED THEN UPDATE
MySQLCREATE TEMPORARY TABLE temp (...)UPDATE target JOIN temp ON ... SET target.col = temp.col
PostgreSQLCREATE TEMP TABLE temp (...)UPDATE target SET ... FROM temp WHERE ...
