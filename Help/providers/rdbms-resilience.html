Resilience Patterns

Polly v8 retry + circuit breaker in RDBSource.Resilience

Pipeline Architecture

Retry Pipeline

Exponential backoff: 1s → 2s → 4s. Max 3 attempts. Only retries transient errors (timeout, network, deadlock).

Circuit Breaker

Opens after 5 failures (50% ratio). Stays open 30s, then half-open for probing. Auto-resets on success.

Health Checks

Database-specific ping queries. Auto-reopens broken connections. 5s timeout per check.

Combined Pipeline

Circuit breaker wraps retry: circuit opens → fast-fail; circuit closed → retry with backoff.

Transient Error Detection

The IsTransientException predicate identifies errors worth retrying:

  • Timeout — query/connection timeout, command timeout
  • Network — transport-level errors, broken pipe, connection reset
  • Deadlock — SQL Server 1205, PostgreSQL 40P01, MySQL 1213
  • Connection — pool exhausted, connection refused, server unavailable
Non-transient errors (syntax errors, constraint violations, permission denied) are NOT retried — they would fail every time.

Usage

Resilient Connection

// Auto-retries transient failures with backoff
await rdbSource.OpenConnectionResilientAsync();
// If circuit is open, throws immediately instead of waiting

Health Check

bool healthy = await rdbSource.CheckConnectionHealthAsync();
if (!healthy) {
    // Circuit is open or connection is dead
    rdbSource.ResetResiliencePolicies(); // Manual reset after fixing
}
