RDBMSDataSource Architecture

Internal design of RDBSource — the core RDBMS plugin with 16 partial class files, 5 helper classes, and support for 8+ database engines.

Class Hierarchy

IRDBSource

Framework interface defining the RDBMS contract: CRUD, schema loading, query execution, DDL generation, and transaction management.

RDBSource

Main implementation — a partial class split across 16 files organized by responsibility. ~5,400 lines total. Database-agnostic via ADO.NET abstraction.

InMemoryRDBSource

Extends RDBSource with IInMemoryDB. Adds ETL-based data sync, lifecycle management (IsCreated, IsLoaded, IsSynced), and structure creation.

RDBDataConnection

Wraps IDbConnection. Handles driver instantiation via DMEEditor.assemblyHandler, connection string building, schema alteration.

Key Design Patterns

Provider Abstraction

Database drivers are loaded dynamically via DMEEditor.assemblyHandler.GetInstance(typeName). The plugin has no compile-time dependency on any specific ADO.NET provider. Supported engines:

EnginePaging SyntaxTemp TablesHealth Check
SQL ServerOFFSET/FETCH#tempSELECT 1
MySQLLIMIT/OFFSETCREATE TEMPORARY TABLESELECT 1
PostgreSQLLIMIT/OFFSETCREATE TEMP TABLESELECT 1
OracleROWNUM subquery—SELECT 1 FROM DUAL
SQLiteLIMIT/OFFSET—SELECT 1
DB2——SELECT 1 FROM SYSIBM.SYSDUMMY1

Resilience Pipeline

Polly v8 ResiliencePipeline combines circuit breaker (50% failure ratio, 30s duration) + retry (exponential backoff, 3 attempts). Health checks issue database-specific ping queries and auto-reopen connections.

Multi-Level Caching

  • Query cache — ConcurrentDictionary<string,string> for compiled query strings
  • Result cache — MemoryCache with configurable TTL (default 5 min, size limit 1024 entries)
  • Prepared statement cache — ConcurrentDictionary for command text reuse
  • Entity structure cache — EntityStructureCache (ConcurrentDictionary with refresh)
  • Property mapping cache — DataStreamer caches reflected property info per type

Bulk Operation Strategies

  • Multi-row INSERT — INSERT INTO t VALUES (...), (...), (...) for SQL Server/MySQL/PostgreSQL/SQLite
  • Temp-table MERGE — Create temp table → bulk insert → MERGE/UPDATE JOIN for SQL Server/MySQL/PostgreSQL
  • Batched fallback — Individual INSERT/UPDATE wrapped in transaction for Oracle/DB2
