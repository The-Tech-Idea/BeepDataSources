DuckDB (in-memory RDB)

Analytic SQL engine with optional file-backed catalog. Project InMemoryDB/DuckDBDataSourceCore/. Class DuckDBDataSource (namespace DuckDBDataSourceCore) inherits InMemoryRDBSource, which inherits RDBSource from DataSourcesPlugins/RDBMSDataSource — so CRUD, schema, Dapper, and transactions follow the same shared RDBMS plugin path as SQL Server or PostgreSQL, with IInMemoryDB structure/load behavior on top. AddinAttribute: category INMEMORY, type DuckDB.

Inheritance & shared plugin

Chain: DuckDBDataSource → InMemoryRDBSource → RDBSource (partial classes under RDBMSDataSource/PartialClasses/RDBSource/). Dialect-specific SQL still flows through IDataSourceHelper for DataSourceType.DuckDB where BeepDM registers templates.

For the full list of engines on RDBSource and how helpers fit in, read RDBMS family overview.

Registration

  • DataSourceType: DuckDB
  • Category: INMEMORY

NuGet (see csproj)

DuckDB.NET.Bindings.Full, DuckDB.NET.Data.Full, plus Beep packages in DuckDBDataSourceCore.csproj.

Delimiters & connection wiring

The constructor sets ColumnDelimiter = "[]" and ParameterDelimiter = "$" (DuckDB-style). It uses RDBDataConnection from the shared plugin and binds ConnectionProperties from the editor by connection name, with DatabaseType = DataSourceType.DuckDB.

File readers

The project registers DuckDB-backed readers for several file types (for example Parquet, JSON, TSV). Registration helpers live in FileReaders/DuckDbFileReaderRegistration.cs (RegisterNonConflicting, RegisterAllPrimaryFormats). NDJSON/JSONL uses DataSourceType.FlatFile so it does not replace the default JSON reader—see in-repo comments on DuckDbNdjsonFileReader.
