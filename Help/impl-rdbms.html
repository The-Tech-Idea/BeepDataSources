Relational (RDBMS) datasources

Shared logic lives in DataSourcesPlugins/RDBMSDataSource (RDBSource partial classes: connection, CRUD, schema, DML helpers, bulk, transactions). Each engine is a thin *DataSourceCore project that inherits RDBSource and sets DataSourceType. Dialect-specific SQL flows through IDataSourceHelper / RDBMS helpers in BeepDM.

Types that inherit RDBSource

All SQL execution, Dapper wiring, schema/CRUD partials, and transaction hooks live on RDBSource in the shared plugin. Engine projects are thin: they set delimiters, optional overrides, and AddinAttribute (DatasourceCategory + DataSourceType).

InMemoryRDBSource (DuckDB & SQLite)

InMemoryRDBSource lives in the same plugin (DataSourcesPlugins/RDBMSDataSource/InMemoryRDBSource.cs): it extends RDBSource and implements IInMemoryDB for structure/load workflows. Two shipped cores build on it:

  • DuckDBDataSource — InMemoryDB/DuckDBDataSourceCore/, DatasourceCategory.INMEMORY, DataSourceType.DuckDB. Uses DuckDB.NET; documented under Local & in-memory.
  • SQLiteDataSource — DataSourcesPluginsCore/SqliteDatasourceCore/, also implements ILocalDB; AddinAttribute category RDBMS, type SqlLite. File lifecycle stays on the local-db path even though SQL behavior comes from the RDBMS stack.

LiteDB does not inherit RDBSource; it is a document store with its own core. See LiteDB.

IDataSourceHelper boundary

Runtime CRUD and metadata on RDBSource delegate dialect work to the helper for the active DataSourceType. Do not duplicate vendor SQL in the core when a helper already exposes the same operation; extend the helper or the shared RDBSource path so all RDBMS drivers stay consistent.

Configuration

Use ConnectionProperties with server fields (Host, Port, Database, SchemaName, Oracle SID/service where applicable), then validate and open. Same editor flow as other datasources (IDMEEditor).

SQLite note

SQLite inherits InMemoryRDBSource (see DuckDB & SQLite above), is registered as RDBMS with DataSourceType.SqlLite, and adds ILocalDB for file lifecycle. See SQLite provider and local & in-memory overview.

Contrast: NOSQL

Document, key-value, and time-series cores use DatasourceCategory.NOSQL and do not inherit RDBSource. See NoSQL overview.

Contrast: REST / SaaS connectors

Vendors under Connectors/ expose HTTP APIs via WebAPIDataSource and typically WebAPIConnectionProperties rather than SQL connection strings. Overview: REST / SaaS connectors; per-vendor command catalogs: flagship conn-* index.
