IDataSource contract

Every datasource implementation in this repository should honor the BeepDM contract in DataManagementModelsStandard. Full detail: BeepDM/.cursor/idatasource/SKILL.md.

Responsibilities

  • Implement connection open/close; keep ConnectionStatus accurate.
  • Use ErrorObject for expected failures; avoid throwing as the default control flow.
  • Keep Entities / EntitiesNames in sync after metadata changes.
  • Delegate dialect-specific SQL to IDataSourceHelper where applicable.

Method areas

Query/execute, CRUD, schema/metadata, transactions — see skill for the full checklist and pitfalls (partial implementations break discovery and UI).

Provider docs

Per-engine topics are under Implementations (phased). REST/SaaS connectors also publish command and model summaries on flagship conn-* pages (REST / SaaS overview).
